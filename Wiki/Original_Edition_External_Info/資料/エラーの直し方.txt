～エラーの直し方～
テキストエディタを使って対象のERBファイルを開いて、書き直すのが基本です

手順：
１　eramaker.exe を起動した時に示されるエラーを直します
	ここでは、主に記号や命令を直すことになります
	たとえば、PRINTFORM が PRINTFOEM になっていたり、行を無視させるための ; が : になっていたりするのを直します

２　era_if_endif_checker　で示された行から調べ初めて、IF と ENDIF のペアを正しくします
	殆どの場合、IF と ELSEIF が逆転しているか、ENDIF が抜けているのを直すことになります
	IF と ENDIF のペアは調教ごとに閉じているのが殆どなので、ひとつの調教について直し終わるごとに era_if_endif_checker で確認するのがよいでしょう
	例えば、eraSQ_uljp00097.zip の eraSQ_2008_08_01_β のERBフォルダで era_if_endif_checker を使うと 「.\EVENT_K09.ERBの1172行目から3264行目の間に終了していないIFがあります。 」と表示されます。この時、1172行目から始まる調教は正常位なので、1172行～1213行の間に確実にひとつはおかしいIF と ENDIF のペアがあることになります。そして、修正できたと思ったならば再び era_if_endif_checker を使って調べます。これ以上ミスがないならば、IF と ENDIF のペアは正しくなっているはずです。

３ 上の二つに該当しないミスを直します。なぜか設定してあるはずの台詞を夢魔が喋ってくれなかったり、PRINT とか喋った時等がこれに該当します
	正直、気付いた人が直すしかありませんし、パターンも多岐なので、よくある例を挙げます
	・誤字脱字
	・「PRINT」と喋る時は、PRINTの後の半角スペースがないことが原因です
	・幾つかの調教において喋ってくれないのは、IF と ENDIF がうまくいっていないときが殆どです。例えば、クンニとアナル愛撫の時だけ何故か喋らないならば、この二つの調教に跨って IF と ENDIF がおかしな具合にペアを作っているのが原因です
	・本来喋るべき台詞を喋らない時は、分岐がうまくいっていないことが多いです。例えば
		IF TALENT:85
			PRINT A
		ELSEIF TALENT:85 && MARK:2 == 3
			PRINT B
		ELSE
			PRINT C
		ENDIF
	という構文だとAを喋る条件よりもBを喋る条件の方が厳しいので、Ｂは喋ってくれません。また、
			P = PALAM:3 + UP:3
			IF P >= PALAMLV:2
				PRINT A
			ELSE
				PRINT B
			ENDIF
	という構文の時に P = PALAM:3 + UP:3 を書き忘れると、Aしか喋りません。
